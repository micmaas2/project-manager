## Proposal 1

**Target file**: `CLAUDE.md`

**Change**: Add a rule under "Workflow Orchestration" (step 4, "Verify before done") stating that every task — including pure git operations (merges, rebases, branch pushes) — must produce a minimal evidence artefact in `artefacts/<task-id>/` before being marked `done`. The minimum artefact is a `verification.md` containing: the command(s) run, their output (or a summary), and a PASS/FAIL verdict against each acceptance criterion.

Proposed addition (insert after the existing step 4 text):

```
**Artefact minimum for git-only tasks**: even when no code is produced, create
`artefacts/<task-id>/verification.md` capturing:
- Commands run (e.g. `git log --oneline`, `ls` of key files)
- Their output
- PASS / FAIL verdict per acceptance criterion
This file is the evidence that satisfies Definition of Done and enables SelfImprover
to run. Tasks with no artefact directory cannot be retrospected.
```

**Rationale**: task-003 was completed informally — no artefact directory, no verification output, no test evidence. When SelfImprover runs post-completion there is nothing to analyse. The process gap is that "merge only" tasks feel too small to warrant an artefact, but the pipeline requires one for traceability and retrospection. Making the minimum bar explicit (a single `verification.md`) closes this gap with minimal overhead.

## Proposal 2

**Target file**: `tasks/queue.json` (schema enforcement, documented in `tasks/queue.schema.json`)

**Change**: Add a `required` constraint for `artefact_path` — it must be set to a non-empty string before `status` can transition to `done`. Document this rule in `queue.schema.json` as an `if/then` constraint:

```json
"if": { "properties": { "status": { "const": "done" } } },
"then": { "required": ["artefact_path"], "properties": { "artefact_path": { "minLength": 1 } } }
```

Also add a note to the "Task tracking" section of `CLAUDE.md`:

```
**Artefact path required on done**: a task may not be set to `status: done` without
a non-empty `artefact_path`. If no code was produced, set the path and create a
`verification.md` (see step 4).
```

**Rationale**: task-003 has `artefact_path` set in queue.json but no directory was ever created on disk. A schema-level constraint (enforced at the point of status update) would catch this earlier. Combined with Proposal 1, it creates a two-layer guard: the schema rejects a `done` transition with no path, and the process rule requires the directory and `verification.md` to actually exist.

**Status**: REQUIRES_HUMAN_APPROVAL
