# Lessons Learned

Post-session retrospective notes. Updated after each agent run or user correction.

| Date | Agent | Lesson | Applied To |
|------|-------|--------|------------|
| 2026-04-05 | Builder | **Verify tool availability before scripting**: `jq` was not installed at build time. Scripts with external dependencies should check for the tool and emit a clear error — and the MVP template should list runtime prerequisites under `tests_required` or a new `prerequisites` field. | All future Builder tasks |
| 2026-04-05 | ProjectManager | **Execute agents sequentially with queue.json as shared state**: The queue.json file is the effective handoff contract between agents — each agent reads status and assigned_to to know what to do. Keeping it updated after every step prevents context drift in multi-step pipelines. | All future pipelines |
| 2026-04-05 | Tester | **Test with fixtures, not just live data**: Running tests against the real queue.json ties test results to live system state. Always test with at least one controlled fixture (empty queue, seeded queue) to make acceptance criteria provable independently of current system state. | All future Tester runs |
| 2026-04-05 | Builder/Reviewer | **List all acceptance-criteria tools in prerequisites**: If an acceptance criterion says "passes shellcheck", then shellcheck must appear in `mvp_template.prerequisites`. The preflight step catches missing tools before Builder runs — but only for tools that are declared. Undeclared tools fail silently during review. | All future tasks with static-analysis criteria |
| 2026-04-07 | Builder | **Use `re.MULTILINE` when searching multi-line content with `^` anchors**: A regex compiled without `re.MULTILINE` treats `^` as start-of-string, not start-of-line. `_BOLD_RE.search(content)` silently returned no matches across all old-format notes until `re.MULTILINE` was added. Always add this flag when the pattern uses `^`/`$` and the input is multi-line text. | All future Builder tasks using regex on file content |
| 2026-04-07 | Builder | **Tighten old-format detection to require identifying fields, not just a heading**: Detecting old-format notes by `# heading` alone matched Dashboard.md and other plain-Markdown files. Adding a second check (at least one `**Key:** value` bold field present) produced false-positive-free detection. When writing format detectors, require the combination of markers that uniquely identifies the format. | All future migration/detection scripts |
| 2026-04-05 | Builder | **Avoid no-op pipeline stages**: A `sed 's/^//'` (replace start-of-line with nothing) has no effect and adds a pointless pipeline stage. Reviewer caught it but it still shipped. Builder should not emit commands that do nothing — if unsure, test the pipeline without the stage first. | All future Builder script tasks |
| 2026-04-07 | ProjectManager | **Check for artefact directory conflicts when assigning task IDs**: artefacts/task-006/ and artefacts/task-007/ were pre-populated from informally completed tasks. When new tasks were queued with the same IDs, paths clashed. Fix: always audit `ls artefacts/` before assigning the next task ID; use a descriptive suffix (e.g. task-007-gate/) for the new task. | All future PM planning sessions |
| 2026-04-07 | SelfImprover | **Documentation-only tasks pass review cleanest when scoped to the minimum viable diff**: task-006 touched exactly 2 lines in CLAUDE.md and 1 field in queue.json — Reviewer approved on first pass with zero change requests. When a task is read-only / docs-only, resist the temptation to refactor adjacent content; minimal diff = no review cycles. | All future docs/CLAUDE.md tasks |
| 2026-04-07 | Builder | **Add skip-if-missing guards to all mandatory session-start steps**: Step 0b (lessons read) includes an explicit "if file does not exist, skip" guard — preventing a hard failure when lessons.md is absent in a fresh repo clone. Any mandatory workflow step that reads a file should include this guard to make the step safe for new environments. | All future CLAUDE.md workflow step additions |
| 2026-04-07 | SelfImprover | **Artefact path conflicts recur — the CLAUDE.md queue section already documents the fix but it was not followed**: The lesson about `ls artefacts/` before task ID assignment was added to CLAUDE.md in the same session it was needed (task-006, task-007). This suggests the instruction must be surfaced at planning time, not just documented. Consider adding it as a preflight checklist item in the PM planning session step. | All future PM planning sessions |
| 2026-04-07 | SelfImprover | **Keep CLAUDE.md and agent YAML format specs in sync**: task-007-gate added an improvement_proposals.md format block to both CLAUDE.md and self-improver.yaml. Reviewer verified consistency explicitly. Any format or protocol defined in CLAUDE.md must be mirrored exactly in the corresponding agent YAML — and the Reviewer checklist should include a cross-file consistency check for formats that appear in multiple locations. | All future tasks touching agent YAML + CLAUDE.md together |
| 2026-04-07 | Tester | **Map every acceptance criterion to a grep-provable file assertion**: task-007-gate achieved 100% pass with zero rework because each criterion referenced a specific file and a specific section verifiable by inspection. For any doc/config task, translate each criterion into a deterministic file-state assertion before marking tests done. | All future documentation and configuration tasks |
| 2026-04-07 | Builder | **Document the vault-sync gap explicitly and queue a follow-up task**: When a capture path ends in a GitHub repo (not the live Obsidian vault), the limitation must be documented in deploy-notes.md AND a follow-up task queued in backlog.md at build time. Leaving the gap undocumented creates a silent data-loss risk for the user who expects live vault availability. | All future capture / sync tasks |
| 2026-04-07 | Builder | **Stdlib-only is a first-class design constraint for laptop scripts**: capture.py used `urllib`/`base64`/`json` only — no pip installs needed. This removes setup friction and dependency drift. For any script that runs on an arbitrary developer laptop, declare "no external dependencies" as an explicit acceptance criterion in the mvp_template. | All future laptop/cross-platform scripts |
| 2026-04-07 | SelfImprover | **Zero reviewer change requests + 100% test pass on first run is achievable with tight scope discipline**: task-008 had no reviewer change requests and 7/7 tests passing. This correlates with an explicit non-scope list, stdlib-only constraint, and two well-bounded capture paths. Tight scope up-front reduces rework — record this as the target baseline for all new tasks. | All future PM planning sessions |

| 2026-04-07 | Builder | **Use `importlib.util.spec_from_file_location` to test hyphenated-filename scripts**: Python cannot `import migrate-vault` directly. Always use `importlib.util.spec_from_file_location("module_name", "/path/to/script.py")` when the script filename contains hyphens. This pattern is required for all Pi4 migration/utility scripts with hyphenated names. | All Builder tasks producing hyphenated-filename scripts |
| 2026-04-07 | Builder | **Thread `--dry-run` through every file-writing function as an explicit parameter**: Migration scripts must pass `dry_run: bool` all the way down to the function that opens files for writing. Handling dry-run only at the top-level caller risks accidental writes from helper functions invoked in non-obvious paths. | All future Builder migration/transformation scripts |
| 2026-04-07 | Security | **Always block private IP ranges in outbound HTTP fetchers**: Scripts that fetch arbitrary URLs from user-controlled data (e.g. vault notes) must block `localhost`, `127.x`, `10.x`, `192.168.x`, and `172.16-31.x` before issuing `requests.get`. Use a hostname-check function pre-request; document the private-IP check in the MVP template security section for any task with outbound HTTP. | All future Builder tasks with outbound HTTP |
| 2026-04-07 | ProjectManager | **Rejected proposal P5 (task-006)**: "add tooling/testing/task-design wording to step 0b" — already present in CLAUDE.md. Check current file state before writing proposals that target existing text. | SelfImprover: read target section before proposing additions |
| 2026-04-07 | ProjectManager | **Rejected proposal P6 (task-007-gate)**: "ls artefacts/ preflight as numbered step" — duplicate of P4 (task-006-lessons Proposal 1), which was approved and applied in the same session. Cross-check proposals across all tasks in a session before presenting to user. | SelfImprover: deduplicate across concurrent task proposals |
