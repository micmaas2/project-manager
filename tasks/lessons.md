| 2026-04-13 | Builder | **Fail-open auth guards require explicit WARNING logging**: When an auth guard fails open (e.g. TELEGRAM_CHAT_ID=None → allow all), add a logger.warning() so operators can detect misconfiguration in logs. Silent fail-open is a MEDIUM security risk — the permissive mode is invisible. | All future Builder tasks with auth guards |
| 2026-04-13 | Builder | **sys.modules pre-injection pattern for Docker-only packages**: When testing a Python module that imports from a package only available in a Docker image (e.g. src.integration), pre-populate sys.modules with MagicMock entries for all transitive deps, then load the file via `importlib.util.spec_from_file_location()`. Canonical example: `artefacts/task-019/test_auth_guard.py`. | Future Builder tasks testing Docker packages |
| 2026-04-13 | Builder | **Docker COPY src/ layer invalidation is deterministic and fast when deps are cached**: When only source code changes (not requirements.txt), the pip install layer stays cached and only the COPY src/ layer rebuilds. | 2026-04-05 | Builder | **Verify tool availability before scripting**: `jq` was not installed at build time. | 2026-04-07 | Builder | **Use `re.MULTILINE` when searching multi-line content with `^` anchors**: A regex compiled without `re.MULTILINE` treats `^` as start-of-string, not start-of-line. `_BOLD_RE.search(content)` silently returned no matches across all old-format notes until `re.MULTILINE` was added. | 2026-04-05 | Builder | **Avoid no-op pipeline stages**: A `sed 's/^//'` (replace start-of-line with nothing) has no effect and adds a pointless pipeline stage. Reviewer caught it but it still shipped. Builder should not emit commands that do nothing — if unsure, test the pipeline without the stage first. | All future Builder script tasks |

| 2026-04-08 | Builder | **n8n credential IDs in workflow JSON templates are placeholders — they must be replaced with real DB IDs after import**: n8n resolves credentials by internal UUID, not by name. Symptoms: auth errors or empty responses in HTTP Request nodes, with downstream nodes never executing. Fix: after deploying any workflow, verify each node's credential ID matches a real credential in n8n (`n8n export:credentials --all`). For workflows that share credentials across re-imports, hard-code the live ID in the canonical source file and document it. | All future n8n workflow deployments |

| 2026-04-10 | Builder | **Cron-deployed shell scripts require a hardening checklist in the MVP template**: task-011 needed 2 CRs and 4 code-quality fixes — all addressing cron-specific gaps (log writability guard, flock concurrency lock, explicit SSH identity, log rotation marked required). | 2026-04-10 | Tester | **`re.MULTILINE` and `re.DOTALL` conflict when `$` is used as a terminator in lazy matches**: combining both flags causes `$` to match end-of-line rather than end-of-string, so a lazy `(.*?)(?=...\|$)` stops at the first newline even though `re.DOTALL` allows `.` to cross line boundaries. This silently truncates multi-line captures to their first line. Fix: drop `re.MULTILINE`; replace `^` with `(?:^\\n)` and trailing `$` with `\\Z`. Reviewer and code-quality-reviewer both missed this — it was only caught by a dedicated multi-line fixture in the test suite. Always add a multi-line fixture when reviewing any `re.DOTALL` regex that uses `$` as a stop anchor. | All future Builder and Reviewer tasks using regex to parse structured LLM output |
